# Настройка авторизации Supabase

1. Откройте проект в Supabase.
2. Перейдите в `SQL Editor`.
3. Выполните файл
   `migrations/20260606190000_auth_roles.sql`.
4. В разделе `Authentication > Providers > Email` оставьте включённым
   провайдер Email.
5. Зарегистрируйте обычного пользователя через мобильное приложение.

Новый аккаунт всегда получает роль `user`. Это защищает систему от
самостоятельного назначения прав администратора.

Чтобы назначить роль администратора уже зарегистрированному аккаунту,
выполните в `SQL Editor`:

```sql
update public.profiles
set role = 'admin', updated_at = timezone('utc', now())
where email = 'admin@example.com';
```

Политики RLS разрешают пользователю видеть и создавать только собственные
заказы. Администратор может читать и изменять все заказы, что будет
использоваться отдельным desktop-приложением.

## Автоматические статусы заказов

Миграция `migrations/20260606195500_automatic_order_statuses.sql` создаёт
справочник из пяти статусов:

1. `Обработка`
2. `Принят`
3. `Готовится`
4. `Готов`
5. `Выдан`

Новый заказ начинает со статуса `Обработка`. PostgreSQL Cron проверяет
заказы каждую секунду, а каждый промежуточный статус длится случайно от
1 до 7 секунд. `Выдан` является конечным статусом.
